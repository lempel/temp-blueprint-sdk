This Package provides careless connection closure mechanism for pure JDBC based DBMS handling.
As you know, ibatis is much more handy but well made raw JDBC implementations are much faster.


Any Servlet(including JSP) never have to close any connections. And You don't have to worry about any Exceptions that may occur. Every connections will be closed automatically .
Just call 'ConnectionHelper.getConnection(String dbmsId)' and forget about 'Connection.close()'.


To faciliate florist.db features:

First, add 'ConnectionListener' and 'ConnectinFilter' to 'web.xml'.
Second, write some properties files and place them into your CLASSPATH.

  • jdbc_pools.properties - list of properties files

        #
        # list JDBC configurations files here.
        #
        # ex)
        # PROP_LIST = /oracle.properties /mysql.properties /postgres.properties
        # or
        # PROP_LIST = /oracle.properties \
        # /oracle.properties \
        # /postgres.properties
        #
        PROP_LIST = /oracle.properties /mysql.properties /postgres.properties

  • [any id].properties - 1:1 definition for JDBC connection pool

        POOL_NAME = XXXX_DB
        CLASS_NAME = oracle.jdbc.OracleDriver
        URL = jdbc:oracle:thin:@127.0.0.1:1521:INT1
        USER = scott
        PASSWORD = tiger
        MAX_ACTIVE = 5
        MAX_IDLE = 1
        MAX_WAIT = 200
        INITIAL = 1
        VALIDATION = select sysdate from dual

    • Now, ready to go.
    See 'ConnectionHelper' for source code level usage.
